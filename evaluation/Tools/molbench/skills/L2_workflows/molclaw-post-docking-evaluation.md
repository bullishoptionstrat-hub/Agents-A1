# Post-Docking Evaluation and Consensus Ranking Workflow

## Applicability

**Use this skill when:** You already have docking result files (poses, scores) from ANY source and need systematic multi-method evaluation, rescoring, interaction analysis, or consensus ranking.

**Do NOT use this skill when:** You have not yet performed docking (use Skill 2). This skill is for **deeper analysis** of existing results.

## Prerequisites

| Input | Source | Required? |
|-------|--------|-----------|
| Docking pose files | Skill 2 output; or external PDBQT/SDF/PDB | Yes |
| Receptor PDB | Skill 1 `prepared_pdb` | Yes |
| Docking scores | From the docking tool | Recommended |
| Numbering scheme info | Skill 1 `numbering_scheme` | Recommended (for residue mapping) |

## Module 1: EquiScore Rescoring

EquiScore is an SE(3)-equivariant graph neural network scoring function, independent of Vina.

**Workflow:** `equiscore_pocket` (extract pocket + split ligand) → `equiscore_screen` (score).

**EquiScore interpretation:** Score range 0–1; higher = better. > 0.7: high-confidence active; 0.4–0.7: moderate; < 0.4: low confidence.

**Multi-pose mode:** For ≤ 50 candidates, use `multi_pose=True, pose_num=3`. For larger sets, use `multi_pose=False`.

**Format note:** If docking results are in PDBQT format, EquiScore's pocket extraction step will generate SDF files. Save the `single_sdf_dir` path — ProLIF will need these SDFs.

**Post-EquiScore download (L3 Principle 14):** Download SDF files generated by EquiScore pocket extraction — these are Category A files needed for ProLIF and user verification.

## Module 2: ProLIF Interaction Fingerprint Analysis

Call `prolif_docking` (for docking poses) or `prolif_pdb` (for crystal/predicted complexes).

**ProLIF detects:** hydrogen bonds (donor/acceptor), hydrophobic contacts, π-π stacking, π-cation, salt bridges, van der Waals contacts, halogen bonds.

### Residue Numbering Mapping for ProLIF (L3 Principle 17 — MANDATORY when task references specific residues)

**MAPPING GATE — Execute BEFORE interpreting ProLIF results:**

1. Determine the numbering scheme of the PDB file given to ProLIF.
2. If the task references residues in a different scheme (e.g., UniProt), build a mapping table.
3. Translate ALL ProLIF residue identifiers to the task's reference scheme before analysis.

Common trap: ProLIF reports residue IDs (e.g., ILE159, VAL149) in the numbering of the input PDB. If the input PDB uses tool-internal sequential numbering (e.g., from Boltz-2), these IDs are NOT UniProt numbers. Searching for "Met793" in ProLIF output from a Boltz-2 structure WILL NOT find it — you must search for the corresponding tool-internal residue number instead.

CORRECT: "ProLIF detected HBAcceptor at ALA145 (Boltz-2 internal #) = Ala719 PDB 1M17 = Ala743 UniProt. This confirms the task-specified Ala743 interaction."
WRONG: "ProLIF did not detect interaction with Ala743" (when Ala743 IS present but numbered differently).

### Post-ProLIF Image Download (L3 Principle 15 — MANDATORY)

Download ALL ProLIF visualization outputs:
- Interaction heatmap (PNG/SVG) — shows which interactions each molecule forms
- Frequency barplot (PNG/SVG) — shows interaction conservation across candidates
- Any other images in the ProLIF output directory

**A ProLIF analysis step is NOT considered complete until all visualization images have been downloaded.**

**Analysis dimensions:**

| Category | Frequency threshold | Interpretation |
|----------|-------------------|----------------|
| Anchor interactions | Present in > 70% of top candidates | Pharmacophore essentials — new designs MUST retain these |
| Differentiating interactions | 30–70% | Structural basis for binding strength differences |
| Unique interactions | < 30% | Individual molecules with special binding modes |

## Module 3: Consensus Ranking

**Step 1 — Data assembly.** Merge all scoring dimensions into a unified matrix.

**⚠ COUNT GATE (L3 Principle 11):** Before assembling the consensus table, verify that the number of molecules in each scoring dimension matches. If EquiScore scored 18 molecules but Vina scored 20, identify and explain the 2 missing molecules (EquiScore failure? Format issue?).

**Step 2 — Normalization.** Each scoring dimension has different scales and directions. Normalize all to 0–1 (higher = better):
- Vina (more negative = better): `norm = (max - x) / (max - min)`
- EquiScore (higher = better): `norm = (x - min) / (max - min)`
- ProLIF interaction count: normalize by maximum observed count
- Boltz-2 probability: already 0–1

**Step 3 — Weighted consensus score.**

| Dimension | Default weight | Rationale |
|-----------|---------------|-----------|
| Vina | 0.30 | Established physics-based scoring |
| EquiScore | 0.30 | ML-based, complementary to Vina |
| ProLIF | 0.25 | Interaction quality, not just total score |
| Boltz-2 | 0.15 | Independent affinity estimate |

If a dimension is missing, redistribute its weight proportionally. Document the rationale.

**Step 4 — Consistency analysis.** For each molecule, compare its rank across methods:
- **Level A (high consistency):** Top 25% by ALL methods → most trustworthy
- **Level B (moderate):** Top 25% by most methods → reliable
- **Level C (divergent):** Methods strongly disagree → inspect docking pose; re-dock if needed

## Module 4: Preliminary SAR Analysis (optional)

When candidates share a common scaffold, combine ProLIF fingerprints with property data for structure-activity insights.

**Computation-first rule (L3 Principle 13):** SAR insights must be grounded in tool-computed data (ProLIF fingerprints, computed property differences, docking score variations). Do NOT generate SAR conclusions from LLM knowledge about drug-target interactions. CORRECT: "Replacing the methyl at C3 with fluoro improved docking by 0.8 kcal/mol; ProLIF shows a new halogen bond with Thr790 (UniProt) = Thr122 (Boltz-2)." WRONG: "Fluorine substitution typically improves binding to kinase hinge regions" (this is LLM knowledge, not a tool-computed observation about THIS specific case).

**Similarity clustering:** Use `calculate_morgan_fingerprint_similarity`. Group molecules with Tanimoto > 0.80 into the same scaffold series.

**Fingerprint differential analysis within a series:** Compare ProLIF fingerprints and docking scores among series members. Identify "what structural change led to what binding change."

**Property-activity correlation:** Check if LogP, MW, or other properties correlate with docking scores.

## Common Failures & Recovery

| Failure | Likely cause | Recovery |
|---------|-------------|----------|
| EquiScore fails on some poses | Ligand format issue; atoms outside pocket box | Re-extract pocket with larger radius |
| ProLIF detects zero interactions | Ligand not in pocket; or format conversion lost coordinates | Verify docking pose; re-convert PDBQT → SDF |
| ProLIF residue IDs don't match task | Numbering scheme mismatch | **Execute residue mapping (L3 Principle 17)** |
| Consensus ranking contradicts all individual methods | Normalization issue with outliers | Remove outliers and re-normalize |

## Quality Gates (Active Checkpoints)

**CHECKPOINT after Module 1 (EquiScore):**
- [ ] EquiScore SDF files downloaded
- [ ] Molecule count matches expectation (any failures explained)

**CHECKPOINT after Module 2 (ProLIF):**
- [ ] All ProLIF images downloaded and verified
- [ ] Residue numbering mapping applied (if task references specific residues)
- [ ] Anchor interactions identified with mapped residue identifiers

**CHECKPOINT after Module 3 (consensus):**
- [ ] Molecule counts are consistent across scoring dimensions
- [ ] EquiScore and Vina agree on ≥ 50% of Top 10
- [ ] Level C divergent molecules examined
- [ ] DiffDock confidence scores NOT used for cross-molecule ranking
- [ ] Consensus weights documented
